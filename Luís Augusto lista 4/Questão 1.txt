1- No windows e linux usa-se o binário para se escrever os arquivos, já mac em alguns arquivoa é utilizado um codigo chamado gray para escreve-los. O computador não consegue entender as letras que nós, seres humanos, utilizamos para escrever. Dessa forma, foi criada uma maneira do computador reconhecer cada símbolo tipográfico usando um esquema de codificação. Tanto o ISO quanto o UTF-8 são esquemas de codificação que através de uma tabela associa cada simbolo tipográfico ao número binário. No caso da ISO ela é um evolução da tabela ASCII que usa 1 byte para repesentar 128 símbolos diferentes (o último bit não era utilizado), já a ISO também usa 1 byte, porém usando todos os 8 bits, consegue representar 256 simbolos diferentes. O UTF-8 faz parte do esquema de codifição chamado unicode que trabalha com multbytes, ou seja, é possivel usar mais de 1 byte para representar os símbolos, no caso do unicode ele consegue representar simbolos com até 32 bits, que é sufiente para representar a maioria dos simbolos existentes, no caso especifico do UTF-8, ele pode utilizar de 1 até 4 bytes para representar os simbolos.
